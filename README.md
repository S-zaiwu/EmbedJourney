## 1 前言

用三个⽉的时间，扎实掌握以 STM32 为核⼼的嵌⼊式软件开发技能，依托人工智能专业背景来打造差异化优势，实现⾼质量实战项⽬，最终拿到心仪的嵌⼊式岗位 Offer。
## 2 大纲

- 掌握嵌入式软件开发（MCU 方向）工程师的基本技能；
- 会独立完成基于 STM32 开发板的项目；
- 会部署 AI 于以 Linux 内核的开发板中。
## 3 计划
### 3.1 启动阶段（约一周时间）
#### 3.1.1 核心目标

- 回顾嵌入式开发所需的算法知识；
- 熟悉 STM32 的开发环境。
#### 3.1.2 基本任务

1. 通过在线 IO 平台，训练基本的算法能力；
2. 根据《stm32 入门教程》搭建开发环境。
#### 3.1.3 检查点

- [ ] 完成牛客网上嵌入式专项训练题库；
- [ ] 能通过程序将 stm32 开发板成功点亮。
### 3.2 基础阶段（约三周时间）
#### 3.2.1 核心目标

- 掌握 STM32 开发所需的基础知识；
- 掌握 RTOS 的基本概念和架构；
- 掌握常见接口的原理和操作。
#### 3.2.2 基本任务

1. 跟着《STM32 入门教程》完成所有学习和实操；
2. 复盘总结相关学习，记录好重难点。
#### 3.2.3 检查点

- [ ] 在 GitHub 上记录每次课的笔记和成果物；
- [ ] 编写对整个基础阶段的思维导图。
### 3.3 强化阶段（约四周时间）
#### 3.3.1 核心目标

- 深入理解 RTOS 的架构；
- 深入理解各个接口的原理；
- 掌握 Linux 开发板的基本知识；
- 了解轻量级人工智能模型。
#### 3.3.2 基本任务

1. 通过**基础阶段**的总结，解决上一阶段遗留下来的问题；
2. 针对重难点，进行多方面、多角度的理解；
3. 回顾 Linux 的基本操作和架构；
4. 学习如何将轻量级 AI 与 Linux 开发板结合的方法。
#### 3.3.3 检查点

- [ ] 对重难点进行详细的分析，并形成相应的文章；
- [ ] 设计系列实验，验证是否对重难点知识有深刻理解；
- [ ] 移植轻量级 AI 到 Linux 开发板上。
### 3.4 项目阶段（约四周时间）（项目暂定）
#### 3.4.1 核心目标

- 实现一个基于 RTOS 开发板的项目；
- 实现一个融合轻量级人工智能模型的 Linux 开发板的项目。
#### 3.4.2 基本任务

1. 实现一个常见的、简单的、融合所学知识的 MCU 项目；
2. 实现一个应用轻量级 AI 的 Linux 项目。
#### 3.4.3 检查点

- [ ] 上传整个项目至 GitHub 上，且配有详细的文档说明；
